# Contributing to GoSkeleton

Thank you for your interest in contributing to GoSkeleton! We appreciate your help in making this project better.

## How to Contribute

### Reporting Bugs
If you find a bug, please check the [issues page](https://github.com/yevhenivashchenko/GoSkeleton/issues) to see if it has already been reported. If not, open a new issue with a clear description and steps to reproduce the problem.

### Feature Requests
We welcome suggestions for new features. Please open an issue to discuss your idea before starting implementation.

### Pull Requests
1. **Fork** the repository and create a new branch for your feature or fix.
2. **Commit** your changes with clear, descriptive messages.
3. **Push** your branch to your forked repository.
4. **Submit a pull request** to the main repository.

## Project Standards

### Code Style
- Follow standard Go conventions (e.g., `gofmt`).
- Keep the project structure clean and modular.
- Ensure that your code maintains the "minimal dependencies" philosophy of this project.

### Documentation
- If you add new features, please update the README to reflect these changes.
- Ensure any new configurations or environment variables are documented.

## Development Setup

1. Clone the repository: `git clone https://github.com/yevhenivashchenko/GoSkeleton.git`
2. Install dependencies: `go mod tidy`
3. Configure your `.env` file based on `.env.example`.
4. Run the project: `go run cmd/server/main.go`
